What is Packet?

Packet is HTTP server software designed to serve dynamic webpages generated in Crisp. As a matter of fact, Crisp/Packet is powering this website right now!

The Packet executable packet.exe is bundled alongside the Crisp interpreter. By default, it is configured to serve files stored in the public-www directory in the same folder. Any folders with a *.csp file extension will be served as dynamic webpages.

Dynamic webpages are Crisp programs with the general format: 

{example}

Packet will pass the parameters in to the program when the page is requested:

  • url - The URL of the requested page.
  • verb - The HTTP verb present in the request, for example "GET" or "POST".
  • post - If the request is a POST request, this will contain the request body URL-encoded.
  • headers - A name-value collection of headers passed up by the client.

The program is expected to evaluate to a proper list of the form: 

(content . (status . (mime . (headers . nil))))

These form the response that will be returned to the client requesting the page:

  • content - The body of the response, represented a string.
  • status - The numeric status code of the response.
  • mime - The MIME type to specify in the response, will be text/html for HTML pages.
  • headers - A name-value collection of headers to pass back to the client.

The respond method provides a convenient way to pass a response object back as a result of the function. For instance, the code that powers this webpage consists of the following: 

{self}
