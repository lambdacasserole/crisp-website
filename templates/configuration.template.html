Configuring Packet

To keep your website as safe and secure as possible, you'll want to be familiar with how to properly configure Packet for your needs. The packet.exe executable reads its configuration from the packet.json file in the same directory. The fields within this file are as follows:

Name Type Description
SpecialFormDirectory string The folder path (relative to the packet.exe binary) to load special form binaries from.
CrispFileExtensions IEnumerable<string> A list of file extensions that should be interpreted as Crisp dynamic pages and not served directly.
InternalServerErrorPage string The file path (relative to the packet.exe binary) of the page to serve to the client in the event of a 500 internal server error.
NotFoundErrorPage string The file path (relative to the packet.exe binary) of the page to serve to the client in the event of a 404 not found error.
DirectoryIndices IEnumerable<string> A list of filenames to serve as directory index (default) files, if present.
DoNotServePatterns IEnumerable<string> A list of regular expressions. Any requests for files matching these regular expressions will be served with a 403 forbidden page instead.
MimeTypeMappings Dictionary<string, string> A map from file extensions to the MIME types they should be served with.

Remember, Crisp/Packet is still in pre-alpha is isn't fully tested or secure! Develop cool stuff, but please be sensible.
